# 1. MVP 目标

MVP 不验证大型三国世界、长期运营或排行榜，而验证以下核心命题：

> 玩家是否愿意在历史大势无法改变的前提下，通过武学成长、装备取舍和江湖抉择，反复体验同一历史篇章并追求不同人物命运？

# 2. 核心假设

## 2.1 武侠感

玩家完成一局后，应首先记住：

- 自己学会了什么；
- 用什么方法赢下或避开交锋；
- 为谁拔剑；
- 放弃了哪段承诺；
- 最终成为了怎样的人。

历史悬疑和史证负责增加判断深度，不承担第一体验。

## 2.2 成长感

一局内至少出现两次明确成长：

1. 确定武学方向；
2. 掌握一个会改变终局行动的新招式。

成长必须通过行动选项体现，不能只有面板数字变化。

## 2.3 重玩感

第二局至少提供以下两项：

- 新武学路线；
- 新装备取舍；
- 不同人物命运目标；
- 另一主角留下的线索或破绽；
- 对上一局事件的新解释。

# 3. 产品范围

## 3.1 MVP-0 纸面原型

- 历史篇章：1 个，《长安残照》；
- 固定主角：2 名，杜缄与任朔；
- 单局时长：15～25 分钟；
- 每局有效选择：8～12 次；
- 作者事件块：约 18～24 个；
- 核心地点：6 个以内；
- 关键原创人物：4 名；
- 普通交锋：每局 2 次；
- 终局决斗或突围：每局 1 次；
- 武学成长节点：每局 2 次；
- 秘籍：2 部角色专属，1 部共享残篇；
- 关键装备：7 件左右；
- 主结局族：4 类（守诺、护人、留录、收押），另有失败变体与武学标签；
- 史证：4～6 份，作为次级内容。

## 3.2 Web 垂直切片

纸面验证通过后实现：

- 移动端场景阅读与 2～5 个行动选项；
- 本局状态、武学、装备、人物关系；
- 条件选项和风险提示；
- 战斗回合与文字反馈；
- 结局、人物命运和人物武谱；
- 本地存档与周目解锁；
- 基础行为统计。

# 4. 单局结构

```text
选择主角与已解锁起始配置
→ 受命或寻人
→ 第一次交锋，确定武学方向
→ 护送、追击与装备取舍
→ 参悟秘籍或接受指点
→ 作出不可逆的恩义选择
→ 终局决斗或突围
→ 结算人物命运、武学与史卷
```

# 5. 角色状态

## 5.1 核心能力与行止

| 能力 | 作用 |
|---|---|
| 武 | 正面交锋、护卫、威慑和突破 |
| 谋 | 观察、判断、布置和识破 |
| 势 | 身份、人脉、军中或府中影响力 |

MVP 不追求复杂升级数值。能力主要作为条件和代价修正。

“义”不作为统一属性，而记录守诺、背诺、护弱、不杀、代人受过等具体行止，并由不同人物分别评价。

## 5.2 局内资源

| 资源 | 作用 |
|---|---|
| 伤势 | 限制武学发挥，影响终局和结算 |
| 内息 | 使用进阶招式、压制伤势或连续行动 |
| 时间 | 决定能否兼顾练功、护送、回营和取证 |
| 风声 | 表示外界警觉，按平静、起疑、暴露、缉拿四级变化 |
| 人情 | 请求指点、治疗、掩护和特殊帮助 |

情报不作为统一数值，而以具体“已知事实或传闻”记录。

## 5.3 生死与自由状态

主角另有两个结局维度：

| 维度 | 状态 |
|---|---|
| 生死 | 生还、死亡、生死不明 |
| 自由 | 自由、落网、收押、脱逃、被迫效力；“被追捕”由风声达到缉拿且仍自由派生 |

风声达到“缉拿”只表示身份已被确认并遭到公开搜捕，不等于已经被捕。只有在场景中被敌方控制，才进入“落网”；落网后若仍存在有意义的选择，可以追加一个短节点处理供词、交换、越狱或主动受押。

# 6. 武学系统

## 6.1 结构

每名主角拥有：

- 1 套基础武学；
- 2 条 MVP 进阶路线；
- 每条路线 2 个可解锁行动；
- 1 个终局标志性用法。

## 6.2 杜缄

### 守剑

- `横剑截流`：守住狭窄通道并保护身后目标；
- `借力退锋`：降低替人挡击造成的伤势；
- 终局用法：一人守巷，为军属争取撤离时间。

### 游剑

- `踏墙换位`：绕过封锁或改变交锋位置；
- `引锋离路`：把追兵从护送目标身边引开；
- 终局用法：越过封锁，从后方破坏追兵队形。

## 6.3 任朔

### 破阵

- `破围`：消耗内息突破多人封锁；
- `震胆`：以军中刀势威慑普通敌人；
- 终局用法：正面打穿一条出城通路。

### 护军

- `回锋护列`：放弃追击，替目标承受攻击；
- `负伤不退`：伤势较重时仍可维持阵线一轮；
- 终局用法：在撤退中组织伤者通过城门。

## 6.4 成长方式

成长来源：

- 实战中符合路线主题的行动；
- 高手指点；
- 秘籍参悟；
- 负伤或失败后的领悟；
- 为保护特定人物产生的突破。

不允许通过重复低风险行动刷熟练度。

# 7. 秘籍系统

## 7.1 学习流程

```text
发现秘籍
→ 判断是否适合当前人物
→ 支付时间、人情或剧情代价
→ 参悟一个方向
→ 在实战中完成掌握
```

持有秘籍不等于已经学会。

## 7.2 MVP 秘籍

| 秘籍 | 适用者 | 功能 |
|---|---|---|
| 《折锋六势》残谱 | 杜缄 | 守剑或卸兵方向，绑定赵衡关系 |
| 《并州军刀撮要》 | 任朔 | 破阵或护军方向，绑定魏石与军营路线 |
| 《调息图诀》残篇 | 两人 | 压制伤势、闭息或终局续战一轮 |

每部秘籍至少存在一种非学习用途，例如交换、托付、引火、归还师门或留给后人。

# 8. 装备系统

## 8.1 槽位

- 兵器 × 1；
- 衣甲 × 1；
- 信物 × 1；
- 消耗品 × 2；
- 剧情物品独立记录。

## 8.2 设计规则

每件重要装备必须：

1. 解锁至少一种行动；
2. 提供一种明确优势；
3. 带来限制、暴露或机会成本；
4. 在至少两个事件中产生作用。

若装备只提供攻击、防御数值，则不进入 MVP。

## 8.3 MVP 装备

| 装备 | 主要作用 | 主要代价 |
|---|---|---|
| 师门短剑 | 隐蔽、巷战、杜缄剑招 | 不利于破甲和长兵器交锋 |
| 旧环首刀 | 强攻、威慑、破坏障碍 | 难以隐藏，使风声趋紧 |
| 并州角弓 | 远程压制和环境互动 | 箭矢有限，近战不能用 |
| 破损札甲 | 抵挡一次重击或箭伤 | 妨碍轻身，暴露军人身份 |
| 百姓夹衣 | 缓和风声、混过盘查 | 防护极低，难藏长兵器 |
| 师门剑印 | 解锁同门对话和指点 | 暴露与王允门客的联系 |
| 同袍箭簇 | 解锁魏石和军中关系 | 暴露任朔身份与旧部关系 |

## 8.4 物品去向

装备与秘籍可以被：

- 使用；
- 损坏；
- 丢弃；
- 交换；
- 赠予；
- 用于断后；
- 带入人物结局。

MVP 不做品质颜色、随机属性、强化、锻造、套装和经济循环。

# 9. 战斗系统

## 9.1 回合结构

每轮依次呈现：

1. 环境、距离和保护目标；
2. 敌方动作与即将发生的威胁；
3. 2～5 个上下文行动；
4. 文字结果；
5. 伤势、内息、位置、风声和关系变化。

## 9.2 可选行动

根据场景动态出现：

- 抢攻；
- 格挡；
- 试探；
- 使用武学；
- 借助地形；
- 使用装备；
- 保护他人；
- 威慑或欺骗；
- 撤退；
- 投降或交换。

不设置固定重复的“普通攻击”按钮。

## 9.3 胜负维度

战斗结果同时判断：

- 是否击败或摆脱对手；
- 是否保护目标；
- 是否负伤；
- 是否错过时间；
- 是否暴露身份；
- 是否被捕或失去自由；
- 是否毁坏装备或证据；
- 是否杀人及其关系后果。

## 9.4 死亡与被捕规则

### 死亡

- 伤势按无伤、轻伤、重伤、濒危、死亡推进；
- 正常情况下，濒危后再次承受重创才会死亡；
- 主动舍命、坠落、火场或正面承受弓弩等行为可以直接致死，但必须显示“致命风险”；
- 死亡不是简单失败，仍然结算救下的人、完成的托付、武学传承和江湖评价。

### 被捕

- 缉拿是外界状态，落网是具体事件，收押且篇章行动结束才构成结局；
- 被捕至少损失兵器、时间、同伴保护、史证或武学机会中的两项；
- 被捕后最多追加一个短节点，避免产生与核心路线等量的新分支；
- 玩家可以主动以自由换取他人通行，也可以在城乱中尝试脱逃；
- 投降可以是聪明选择，但不能成为无代价的安全捷径。

# 10. 选择规范

一个有效选择至少改变以下两项中的一项，并优先改变两项：

- 资源；
- 武学成长；
- 装备；
- 人物关系；
- 路线；
- 人物命运；
- 已知信息；
- 延迟后果。

重大选择应体现不可兼得，例如：

- 练功还是赶路；
- 保存秘籍还是救人；
- 穿甲强闯还是换衣潜行；
- 完成命令还是尊重被护送者的选择；
- 包庇同袍还是阻止其作恶；
- 杀死高手夺谱还是留其性命换取指点。

# 11. 多周目成长

## 11.1 人物武谱

每名主角独立记录：

- 已发现的武学路线；
- 掌握过的招式；
- 秘籍完整度；
- 使用过的重要装备；
- 武学结局和挑战称号。

再次进入该人物时，可以选择已解锁且符合其经历的初始配置。

## 11.2 江湖见闻

跨人物共享的是信息，而不是等级和装备仓库：

- 某位高手的身份和破绽；
- 某处秘籍或兵器的下落；
- 某招的破解线索；
- 新的相遇和对话条件。

另一人物仍需具备相应武学，才能利用这些信息。

# 12. 结算

每局结算展示：

1. 主角结局族；
2. 主角生死与自由状态；
3. 关键人物命运；
4. 本局武学路线与已悟招式；
5. 秘籍和装备去向；
6. 新增江湖见闻或史证；
7. 武学标签与挑战完成情况；
8. 下一局可尝试的明确线索。

# 13. 重玩动力

优先级为：

1. 尝试另一条武学路线和终局打法；
2. 改变人物命运与恩义选择；
3. 收集秘籍、装备经历与武学标签；
4. 从另一人物视角理解历史；
5. 完成不杀、低伤、基础武功等挑战；
6. 正式排行榜，MVP 后置。

# 14. 数据与测试指标

Web MVP 至少记录：

- 各选项选择率；
- 武学路线选择率；
- 招式在后续事件中的使用率；
- 装备携带、交换和放弃率；
- 玩家开始第二周目的比例；
- 结束到重开的时间；
- 第二周目选择同一主角还是另一主角；
- 各结局和人物命运达成率；
- 玩家放弃位置；
- 战斗中无法理解局势的反馈。

# 15. MVP 通过标准

满足以下条件后再扩展内容：

- 多数测试者将体验描述为“三国武侠”，而非“历史破案”；
- 玩家能说出自己本局形成了哪种武学风格；
- 至少一个新招式在获得后改变了实际选择；
- 不同武学路线产生不同终局策略；
- 没有一件装备或一条路线成为无条件最优；
- 玩家能说清一次重要取舍的收益与代价；
- 玩家理解“缉拿”和“被捕”的区别，且死亡或收押不被视为突然的系统惩罚；
- 有明显比例的玩家主动开始第二局；
- 第二局至少带来一种新打法和一种新认知。

# 16. 当前不做

- 大型开放世界；
- 自由刷怪和挂机练功；
- 数十门武学与复杂境界；
- 随机装备掉落；
- 品质颜色、强化、锻造与套装；
- 商店经济；
- 多人在线；
- 正式排行榜；
- 完全自由的 AI 行动；
- 付费系统。

# 17. 实施顺序

1. 已依据 `docs/SYSTEM_DESIGN.md` 完成杜缄代表路线与边界状态验证，结论见 `docs/DU_JIAN_MECHANICS_VALIDATION.md`；
2. 编写 `CONTENT_SCHEMA.md`，并将杜缄路线转换为 YAML 或 JSON；
3. 对全部选项组合执行可达性、资源、装备、成长和结局自动校验；
4. 完成任朔交叉路线与双人物状态校验；
5. 进行 5～10 人纸面测试；
6. 根据结果删减无效系统；
7. 开发移动端 Web 垂直切片。
