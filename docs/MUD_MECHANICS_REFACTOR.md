# 1. 重构目的

旧版核心体验是“阅读场景—选择选项—进入下一场景”，虽然具备分支、资源和结局，但缺少玩家可重复使用的动词、可查看状态、空间探索、主动训练和多回合战斗，因此更接近分支小说而非 MUD。

本轮重构的目标不是复制传统 MUD 的命令记忆、刷怪和挂机，而是建立以下循环：

```text
观察地点
→ 自由选择交谈、搜索、训练、调息、移动或迎战
→ 消耗体力与局势时间
→ 获得阅历、熟练、物品、情报或伤势
→ 查看状态、行囊、武学和行动记录
→ 返回剧情并以新的能力解决历史事件
```

剧情事件继续决定价值取舍和人物命运；MUD 内核负责过程中的操作感、成长感和空间感。

# 2. 已确认的产品决策

## 2.1 操作方式

采用“上下文按钮 + 可选文字命令”的混合形式。

- 玩家不输入命令也能完成所有核心操作；
- 熟悉 MUD 的玩家可以直接输入命令；
- 按钮和命令调用同一规则函数，不存在两套结算；
- 中文与英文命令使用别名映射。

## 2.2 成长边界

- 等级、气血、体力和熟练度属于局内状态；
- 新周目不继承无限叠加的数值；
- 特殊实战可以永久解锁新招式或起始配置；
- 永久成长扩大方法，不直接碾压历史场景。

## 2.3 敌人与练级

采用有限敌人、有限训练、实战突破：

- 敌人被击败后不会原地刷新；
- 每个地点最多进行两次有效专项练习；
- 第二次训练收益递减；
- 局势压力达到阈值后不能继续练功；
- 失败会造成伤势和时间损失，但不会清空所有进度；
- 不提供挂机、自动战斗和无限经验来源。

# 3. 玩家可用动词

| 类别 | 按钮 | 文字命令示例 |
|---|---|---|
| 环境 | 观察、搜索 | `look`、`search`、`观察`、`搜索` |
| 人物 | 交谈 | `talk 赵衡`、`问 阿萝` |
| 移动 | 前往相邻地点 | `go 西市废仓`、`去 军属里巷` |
| 成长 | 练剑、练刀、调息 | `practice`、`练剑`、`调息` |
| 状态 | 状态、行囊、武学、记录 | `status`、`inventory`、`skills`、`记录` |
| 战斗 | 迎战、试探、攻击、格挡、地形、撤退 | `fight`、`probe`、`attack`、`defend`、`flee` |
| 帮助 | 命令说明 | `help`、`帮助` |

命令输入错误时只提供可理解的反馈，不消耗资源。

# 4. 世界与地点

## 4.1 地点图

当前垂直切片包含：

```text
王允府外巷
  ↕
西市废仓 ↔ 王允府旧宅
  ↕
军属里巷 ↔ 后院 ↔ 军营小路 ↔ 并州军营
  ↕
废寺偏殿 ↔ 废寺山门 ↔ 里巷出口 ↔ 长安西门 ↔ 城门内侧
```

每个地点至少定义：

- 基础描述；
- 当前人物；
- 相邻出口；
- 一次性搜索结果；
- 可用训练环境；
- 有限敌人或特殊遭遇；
- 与剧情事件的地点映射。

## 4.2 剧情与自由行动

剧情页面提供“江湖行动”和“状态”入口。进入自由行动时，MUD 位置与当前剧情地点同步。

自由行动不会替代剧情：

- 返回剧情后仍从原事件继续；
- 剧情可以读取 MUD 层获得的招式、伤势和物品；
- 后续版本将允许地点状态反向改变剧情文本与选项；
- 历史锚点仍由剧情事件推进，不由刷怪次数决定。

# 5. 局内状态

## 5.1 角色状态

| 状态 | 作用 |
|---|---|
| 等级 | 表示本局实战层次，当前上限 3 |
| 阅历 | 来自首次搜索、有效训练和有限战斗 |
| 气血 | 战斗中的短期承受能力 |
| 体力 | 移动、训练和战斗动作消耗 |
| 内息 | 施展进阶招式，继续与剧情资源共用 |
| 伤势 | 战败和重创造成的长期限制 |
| 熟练度 | 记录短剑或环首刀的局内掌握程度 |
| 局势压力 | 限制可用于自由行动和练功的窗口 |

## 5.2 当前阈值

- 一级升二级：30 阅历；
- 二级升三级：75 阅历；
- 等级上限：3；
- 熟练 30：解锁“截腕”；
- 熟练 60：进入精通阶段，后续扩展组合技；
- 熟练 100：化境，仅作为高难挑战目标；
- 局势压力 6：停止有效练功；
- 局势压力 7：无法安心调息；
- 局势压力 8：必须回归剧情或处理终局。

# 6. 训练与反刷取

每个地点分别记录训练次数。

```text
第一次有效训练：熟练 +9，阅历 +5
第二次校正训练：熟练 +4，阅历 +2
第三次及以后：无收益
```

训练消耗 2 体力并推进局势压力。不同地点未来可以支持不同训练主题：

- 门框：守势与距离；
- 断墙：轻身与换位；
- 军营：刀法和队列；
- 巷口：狭窄地形；
- 伤兵处：负伤运功与护列。

调息每个地点最多有效两次，恢复体力与一口内息，同样推进局势。

# 7. 多回合战斗

## 7.1 战斗状态

- 玩家气血、体力、内息；
- 敌方气力；
- 敌方架势；
- 当前轮次；
- 敌方公开意图；
- 地点环境；
- 可撤退路线。

## 7.2 敌方意图

当前使用确定性循环：

```text
试探逼近
→ 收刀护身
→ 蓄力重击
→ 抢步突刺
```

玩家可以依据意图选择：

- 试探削弱架势；
- 快攻造成气力伤害；
- 格挡减伤并恢复体力；
- 使用地形同时伤害与破势；
- 消耗内息施展招式；
- 主动撤退。

关键战斗不使用隐藏随机命中率。

## 7.3 胜负

胜利条件：

- 敌方气力归零；或
- 敌方架势崩溃且气力已经很低。

战败结果：

- 主角伤势上升；
- 气血恢复至可继续行动的低位；
- 局势压力额外推进；
- 敌人保留，不通过刷新制造新敌人。

# 8. 跨周目解锁

当前示例：在西市废仓击败劫掠者后永久解锁“疾步出鞘”。

永久记录格式：

```json
{
  "techniques": ["mud.quick_draw"]
}
```

下一周目开局即可使用该招，但仍需支付内息，并受敌方意图和场景条件限制。

后续可扩展：

- 已发现的起始招式；
- 起始信物与有限装备配置；
- 特殊身份入口；
- 挑战条件；
- 新对话和训练师。

# 9. UI 结构

MUD 页面包含：

1. 当前地点与局势压力；
2. 等级、气血、体力、内息；
3. 地点描述、在场人物和行动日志；
4. 上下文动作按钮；
5. 文字命令输入；
6. 江湖、状态、行囊、武学、记录、剧情六个标签。

所有核心触控按钮目标高度不低于 42～44 像素。

# 10. 数据和存档

MUD 单局状态与原剧情存档一起保存：

- 等级和阅历；
- 气血与体力；
- 地点与世界行动次数；
- 局势压力；
- 熟练和训练次数；
- 已搜索地点；
- 已击败敌人；
- 已悟招式；
- 当前战斗；
- 行动日志。

永久招式独立保存在人物武谱元数据中。旧存档没有 MUD 字段时自动创建默认单局状态。

# 12. 验证标准

- 不输入命令也能完成探索、训练和战斗；
- 命令与按钮产生完全一致的状态结果；
- 状态、行囊、武学和记录随时可查看；
- 同一地点不能无限获得训练收益；
- 已击败敌人不会刷新；
- 战斗至少需要两种不同动作；
- 战败不会清空剧情进度；
- 永久解锁不会提供无限数值增长；
- 返回剧情后事件位置和存档保持正确；
- 浏览器控制台、请求和页面脚本无错误。
