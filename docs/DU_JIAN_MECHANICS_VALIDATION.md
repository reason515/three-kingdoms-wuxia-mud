# 1. 验证目标

本轮验证不评价文风和剧情感染力，只检查 `SYSTEM_DESIGN.md` 的规则能否把 `DU_JIAN_ROUTE.md` 中的选择稳定推导到合法且有差异的结果。

主要问题：

1. 守剑、游剑、守令和被捕路线是否都可完成；
2. 余时、内息、伤势和风声是否越界；
3. 玩家是否会遇到无可用选项但又没有结局的死路；
4. 物品持有、托付、损坏和没收是否自洽；
5. 缉拿、落网、收押和脱逃是否严格区分；
6. 武学与结局是否可达且判定唯一；
7. 是否存在无代价的完美路线或明显支配其他路线的选择。

# 2. 验证方法

采用两层验证：

- **人工逐节点推演**：检查行动条件、机会成本、人物命运和叙事因果；
- **临时确定性脚本**：将五条代表路线写成最小状态转换，检查资源边界、互斥状态、生死状态和结局优先级。

验证脚本：

```text
tools/validate_du_jian_routes.py
```

运行命令：

```bash
PYTHONIOENCODING=utf-8 python tools/validate_du_jian_routes.py
```

该脚本不是正式内容引擎，也不替代后续 YAML 和完整可达图校验。它只把本轮纸面推演中的关键假设变成可重复执行的回归测试。

# 3. 本轮采用的统一判定

## 3.1 资源与状态

- 余时和内息不足时，不能先扣成负数再规范化；选项应直接锁定。
- 伤势从濒危再次上升时变为死亡。
- 风声达到缉拿时，若自由仍为自由，只派生“被追捕”，不会自动落网。
- 落网后余时立即归零；续段结束后只能进入收押、脱逃或死亡。
- 守剑和游剑主路线互斥。

## 3.2 物品

- `manual_carried` 与 `manual_entrusted` 互斥。
- 杜缄的止血散与阿萝药囊分别记录。
- 被没收物仍存在于场景中，但不能作为玩家当前持有物使用。
- 脱逃时未取回的被没收物统一转为遗失。

## 3.3 主结局优先级

```text
囚剑长安
→ 负刃护人
→ 残剑留录
→ 守诺成刃
→ 城下无名
```

生死、自由、人物命运、传承和行止是附加维度，不与主结局族争抢同一个枚举。

# 7. 路线平衡结论

## 7.1 没有发现无代价完美路线

五条代表路线均至少失去一项重要内容：

- 守剑：余时、内息、健康、短剑和师门关系；
- 游剑：许简的终局作用、多数病患、剑印和短剑；
- 守令：阿萝、老人、完整证据和本局成长；
- 收押：全部可见装备、证据、余时和自由；
- 脱逃：全部被没收物、身份、承诺和低风声状态。

## 7.2 暂未形成单一路线支配

- 守剑在人物保护上最稳定，但身体和资源代价最大；
- 游剑最容易保全自身，但不能独自搬运和保护所有人；
- 守令最容易完成，但人物命运和可重玩内容最差；
- 投降可以换人命，但会失去至少两项重要资源和行动权；
- 脱逃恢复自由，但不会取回被没收的兵器、证据和残谱。

## 7.3 成长机制通过初测

- 守剑和游剑都能在 D07 成招；
- 两个招式都能在 D09 和 D11 使用；
- 放弃成招能保留余时或形成传承；
- 穿札甲会关闭游剑成招，代价明确；
- 两条主路线不能在同一局同时成立。

# 8. 尚未解决的风险

本轮结论为“代表路线通过”，不是“全部内容已经证明正确”。以下内容必须在结构化数据阶段继续验证：

1. D01～D12 所有选项组合的完整可达图；
2. 许简独自携带残页后的持有者与遗失状态；
3. 阿萝未同行、风声达到缉拿时的调度优先级；
4. D11 三轮战斗中提前脱离与第三轮选项的显示规则；
5. 死亡后主结局族、人物命运和装备去向的组合文本；
6. 每件关键装备是否确实影响至少两个事件；
7. D06 同一事件连续两次选择在移动端的节奏是否过长；
8. 赵衡态度变化尚需全部转换为结构化效果，不能继续使用“偏信任”等文本状态；
9. `original_leaf` 等内容应在正式 schema 中建模为物品持有者和物品状态，而不是只用布尔标记。

# 9. 验证结论

当前杜缄路线的规则内核可以支持：

- 两条差异明确的武学路线；
- 履令、背令、投降、收押和脱逃；
- 独立的伤势、风声、生死与自由状态；
- 有代价的装备、秘籍和证据去向；
- 确定性结局判定。

**结论：有条件通过，可以进入内容 Schema 设计和全路线自动校验阶段。**

进入任朔路线之前，应继续完成：

1. 内容 Schema、机器校验文件和首版引用校验器已经完成；
2. 将杜缄路线转换为 JSON；
3. 增加状态空间遍历，校验资源、互斥状态与结局可达性；
4. 将本文件中的五条代表路线转为读取真实内容数据的正式回归测试。
