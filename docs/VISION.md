# 1. 项目一句话

《汉末江湖录》是一款移动端优先、选择驱动、可反复挑战的三国武侠文字 RPG。玩家以无名小人物的身份进入真实历史事件的夹缝，在不可逆的取舍中调查隐秘、结交人物、修习武艺，并决定那些未被史书记载之人的命运。

# 2. 核心愿景

游戏要让玩家产生三种感受：

> 我身处一个正在真实发生的汉末乱世。
>
> 我无法轻易改变天下大势，但我的每个决定都会让一些人的命运不同。
>
> 我这一局看到的未必是真相，我还想从另一个身份和立场再走一次。

游戏以有限而精炼的历史篇章为内容单位，通过身份、路线、人物关系、能力构筑、情报真伪和挑战条件的组合，形成可持续重复游玩的体验。

# 3. 产品定位

《汉末江湖录》是：

- **移动端优先的选择型文字 RPG**：主要操作是在当前情境下从 2～5 个行动中作出选择。
- **周目制叙事游戏**：每个历史篇章可以反复进入，每次历程约 15～30 分钟，并产生独立结局与人物小传。
- **三国历史夹缝中的武侠故事**：史实构成不可轻易改变的骨架，虚构人物和江湖势力活动于史书留白之处。
- **保留 MUD 表现力的规则制世界**：地点、人物、物品、伤势、武学、阵营、情报和事件均由明确状态驱动。
- **以求真和人物命运推动重玩的游戏**：排行榜是高阶挑战的放大器，而不是唯一动力。

# 4. 游戏不是什么

《汉末江湖录》不是：

- **不是传统命令行 MUD**：不要求玩家记忆或输入固定指令。
- **不是以自由输入为核心的 AI 跑团**：关键剧情和结果必须受规则约束。
- **不是普通视觉小说**：选择会持续改变资源、能力、关系、身份暴露和后续行动空间。
- **不是三国宏观战争模拟器**：玩家首先是历史洪流中的小人物，而不是一开始便统率天下的君主。
- **不是无限生成剧情**：重复性来自优质有限事件的条件组合、视角差异和状态联动。
- **不是依赖永久数值碾压的挂机游戏**：局外成长主要解锁新的可能性，而不是不断降低挑战。

# 5. 核心设计支柱

## 5.1 历史夹缝

重要史实是历史锚点，原则上保持可辨认的结果；玩家主要介入史书没有详述的过程、动机和局部命运。

玩家可能无法阻止长安陷落，却可以决定一份名册落入谁手、一群百姓能否逃生、某位无名密使是否被记作叛徒，以及后世最终相信哪个版本。

核心原则：

> 大势或不可改，人心自有去处。

## 5.2 江湖抉择

操作可以简单，决策不能轻薄。每个有效选择至少应改变状态、消耗资源、影响关系、暴露身份、关闭路线或留下延迟后果中的一项。

游戏不追求频繁制造生死转折，而追求稳定的机会成本：玩家选择成为什么样的人，也意味着放弃另一些可能。

## 5.3 多周目求真

单次通关只能看到一个身份和立场下的局部真相。不同路线提供相互矛盾的证词、密信、军令、传闻和人物动机。

玩家通过反复游玩收集“史证”，逐步补全事件全貌，并最终决定如何在自己的《汉末江湖录》中记录这段历史。

## 5.4 人物命运

历史人物不只是任务发布者，虚构人物也不只是剧情工具。人物应拥有公开目标、私人牵挂、政治立场和必须付出的代价。

结算不只报告主角结局，还应展示关键人物的生还、死亡、归属、关系、名声与未解命运。

## 5.5 文字沉浸

文字不是对画面的廉价替代，而是核心表现形式。

场景和战斗参考优秀文字 MUD 的表达能力，以简洁、有动作感的文字呈现环境、距离、招式、伤势和局势变化。系统反馈应融入叙事，而不是堆积冰冷数值。

## 5.6 简单操作，系统深度

移动端核心操作保持在 2～5 个上下文选项之间。深度来自：

- 稀缺资源的取舍；
- 武学和身份解锁的行动方式；
- 情报真假与信息不完整；
- 人物关系和阵营立场；
- 延迟后果与不可逆承诺；
- 不同难度和挑战条件。

# 6. 重复游玩的动力层级

重复挑战按以下优先级建立：

1. **真相拼图**：从不同身份和阵营补全历史事件背后的秘密。
2. **人物命运图谱**：探索关键人物不同的生死、归属与关系。
3. **路线与能力构筑**：使用不同武学、专长和资源解决同一困局。
4. **结局与江湖评语**：收集主结局、特殊称号和差异化人物小传。
5. **历史挑战与排行榜**：在统一种子和特殊限制下竞争表现。

排行榜不设置唯一价值标准，应按篇章、身份和挑战条件拆分，例如无伤、低暴露、救援、情报完整度和极限难度榜。

# 7. 成长方向

## 7.1 局内成长

第一版建议采用少量能力和资源：

- 能力：武、谋、义、势；
- 资源：时间、伤势、疑迹、人情、情报；
- 专长：剑术、轻功、医术、辩才、谍术、兵法等。

专长应优先解锁新的行动选项，而非只提供数值加成。

## 7.2 局外成长

局外成长以扩展选择空间为主：

- 新身份与出身；
- 新武学和专长；
- 新事件和人物视角；
- 初始信物与情报；
- 更高难度和特殊挑战；
- 江湖史卷中的新证据与注解。

# 8. 首个篇章方向

首个垂直切片暂定为《长安残照》：

- 时间：初平三年（192 年）；
- 历史背景：董卓被诛，王允掌权，李傕、郭汜即将反攻长安；
- 固定大势：长安陷落、王允身死、吕布败走；
- 玩家空间：密信与名册的归属、无名者的生死、情报的真假、局部救援与历史记录；
- 单局时长：15～30 分钟；
- 初始身份：3 种；
- 关键人物：5～7 名；
- 主结局：6 个以上；
- 核心事件：约 30 个；
- 主要目标：验证玩家通关后是否愿意立刻以另一种身份再走一次。

# 9. 设计判断标准

出现设计分歧时，依次判断：

1. 是否增强了玩家身处汉末历史现场的感觉？
2. 是否制造了有信息依据的真实取舍？
3. 是否改变后续状态、路线、关系或认知？
4. 是否为重复游玩提供了新的视角，而不是重复劳动？
5. 是否能通过移动端 2～5 个选项清晰完成？
6. 是否适合先做成小而完整的历史篇章？

如果某项设计只增加操作和数值，却没有增强历史感、选择、人物命运或求真动力，应推迟或删除。
