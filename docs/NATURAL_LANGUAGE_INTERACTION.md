# 1. 设计定位

《汉末江湖录》采用 **上下文选项为主、自然语言为辅** 的交互方式。

玩家在每个情境中从 2～5 个行动选项中作出决定。选项负责完整承载主线探索、人物互动、战斗和结局流程，保证玩家能够在移动端单手完成游戏。

自然语言输入不再是核心玩法，仅作为非必要的增强功能，用于帮助、回顾、闲谈或实验性交互。它不能绕过规则，也不能产生排行榜模式中的额外优势。

# 2. 交互目标

核心交互应满足：

- 无需记忆命令；
- 无需频繁调用手机键盘；
- 每次只面对少量、清晰、彼此有差异的行动；
- 选择前能够判断主要收益、风险或条件；
- 选择后立即看到叙事反馈与状态变化；
- 任何关键剧情都可以只通过点击完成。

# 3. 基础交互循环

```text
阅读场景与局势
  ↓
查看2～5个上下文行动
  ↓
判断条件、收益与风险
  ↓
作出不可撤销的选择
  ↓
规则系统更新状态
  ↓
以叙事文字呈现结果
  ↓
进入新的情境
```

# 4. 选项设计规范

## 4.1 每个选项必须改变局面

一个有效选项至少应造成以下一项变化：

- 获得或消耗资源；
- 改变人物关系；
- 改变阵营立场；
- 增加身份暴露或疑迹；
- 获得可能为真的情报；
- 关闭或开启后续路线；
- 为未来事件留下延迟后果。

不设置只换一句文本、但对状态和认知均无影响的伪选择。

## 4.2 每个选项必须存在机会成本

优先使用以下稀缺资源制造取舍：

| 资源 | 作用 |
|---|---|
| 时间 | 决定能否兼顾多个目标，部分事件会随时间失效 |
| 伤势 | 限制战斗、赶路和运功，强行动会加重伤势 |
| 疑迹 | 表示身份暴露和敌方警觉，过高会改变事件与结局 |
| 人情 | 请求人物帮助或调用关系，每局获取和使用均有限 |
| 情报 | 用于判断、交换、欺骗和解锁行动，但可能不完整或失真 |

玩家通常不能同时保全任务、人物、名声和自身安全。

## 4.3 重要但公平

玩家可以不知道确切结果，但应能判断风险类型。系统应通过场景文本、人物神态、时间提示和选项标签提供依据，避免毫无征兆的恶意陷阱。

推荐格式：

```text
【拆开密信检查】
可能获得：关键信息
主要代价：破坏封蜡，降低收信人的信任
```

在需要保留悬念时，可以隐藏具体数值，但不能隐藏玩家角色理应知道的事实。

## 4.4 选项数量

- 普通情境：2～3 项；
- 复杂抉择：3～4 项；
- 少数阶段终局：最多 5 项；
- 不以凑数方式增加选项；
- 条件选项可以显示为锁定状态，用于提示其他构筑的可能性。

锁定选项示例：

```text
【为伤者辨明毒性】
需要：医术
```

这类提示能够自然推动玩家在下一周目尝试新路线。

# 5. 选项类型

## 5.1 行动方式

同一目标可通过武力、潜行、交涉、医术、情报或人情解决。

## 5.2 价值立场

在守诺与求生、个人与大局、复仇与公义、门派与国家之间作出选择。

## 5.3 信息判断

玩家根据不完整或相互矛盾的证据，决定相信谁、传递什么、隐瞒什么。

## 5.4 不可逆承诺

投靠阵营、公开身份、牺牲证据、杀死人物等行动会关闭部分路线，并在后续持续产生影响。

# 6. 战斗交互

战斗采用情境化选择，不设置反复点击“普通攻击”的循环。每场重要战斗建议控制在 3～6 轮。

每轮应先说明：

1. 当前环境与距离；
2. 敌我动作和伤势；
3. 局势发生了什么变化；
4. 玩家当前可以采取哪些行动。

可用选项包括：

- 抢攻；
- 守势；
- 试探；
- 借助地形；
- 保护同伴；
- 攻击弱点；
- 佯败或欺骗；
- 撤离或投降。

战斗选择必须同时服务于叙事。玩家可能赢得交锋却失去证人，也可能负伤撤退却保住关键情报。

# 7. 叙事反馈

选择后的反馈由三部分组成：

1. **画面**：用环境和动作建立空间感；
2. **结果**：说明人物行为及即时后果；
3. **状态**：以简洁方式提示伤势、关系、资源或线索变化。

示例：

> 刀锋擦过梁柱，震落一蓬积灰。你侧身让过半步，剑尖顺着对方腕骨向上挑去。那汉子强行收刀，虎口已被震裂，却借着退势堵住了门口。院外，又响起第二个人的脚步。
>
> **局势变化：**去路被封；援兵将在下一轮抵达。

状态提示应帮助判断，不应取代文字表现。

# 8. 辅助导航

移动端常驻或可快速展开的功能包括：

- 当前目标；
- 人物状态；
- 武学与专长；
- 行囊；
- 人物关系；
- 已知情报；
- 江湖史卷；
- 本局行动回顾。

关键内容不能隐藏在多层菜单中。场景文本和行动选项始终拥有最高视觉优先级。

# 9. 自然语言增强

自然语言可以在后续版本用于：

- 询问“我为什么会到这里”；
- 回顾已知人物和线索；
- 以角色口吻获得规则帮助；
- 与非关键 NPC 进行补充闲谈；
- 将玩家表达匹配到当前已有选项。

自然语言不得：

- 创造不存在的物品或人物；
- 绕过能力、资源和剧情条件；
- 改写关键历史结果；
- 为排行榜玩家提供额外行动；
- 取代预先设计的重要选择。

# 10. 设计结论

《汉末江湖录》的交互原则是：

> **减少输入，不减少判断；减少按钮，不减少代价。**

玩家不需要学习如何操作游戏，但必须认真决定自己愿意付出什么、相信什么，以及想成为什么样的人。
