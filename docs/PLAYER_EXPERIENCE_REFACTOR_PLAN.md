# 1. 文档目的

本文汇总当前试玩暴露出的玩家体验问题，并定义下一轮重构方向。

当前版本已经具备剧情分支、历史背景、自由探索、命令输入、有限训练、多回合战斗和状态面板，但这些能力以多个相互独立的界面和规则入口存在。玩家虽然“可以做很多事”，却难以理解：

- 当前目标是什么；
- 为什么要采取某个行动；
- 剧情和自由行动是什么关系；
- 每项状态会带来什么影响；
- 信息按照什么时间顺序出现；
- 练功和战斗如何改变人物命运。

重构目标不是继续增加功能，而是把已有功能组织成一个清楚、连续、可预测的游戏循环。

## 1.1 当前实施状态

P0 第一批已经启动：

- [x] 剧情页增加持续可见的当前任务和建议下一步；
- [x] 主界面精简为气血、体力、内息和城门进度；
- [x] 状态可点击查看含义、当前影响和下一阶段；
- [x] 观察、搜索、交谈、训练、调息和战斗接入剧情页；
- [x] 文字命令改为默认折叠的“更多操作”；
- [x] 剧情选择结果取消覆盖弹层，改为正文下方时间线；
- [x] 时间线统一为旧信息在上、最新信息在下；
- [x] 旧存档即使停留在独立 MUD 页面，也统一恢复到剧情主界面；
- [x] 将西市伏击替换为剧情绑定的多回合战斗，胜利、战败和撤退分别写回后续场景；
- [ ] 让训练、战斗、伤势和物品在更多剧情节点改变条件（西市伏击已接入）；
- [ ] 删除过渡期独立 MUD 页面代码；
- [ ] 从内部规则上合并余时、局势压力和城门进度；
- [ ] 完成地点阶段开放和 NPC 日程。

当前批次的目标是先验证统一界面的可理解性，不把过渡实现误认为最终架构。

# 2. 核心诊断

## 2.1 当前仍是两套游戏

现有体验由两个平行部分组成：

1. **剧情模式**：阅读长段文本，在若干关键选项中取舍；
2. **江湖行动模式**：移动、搜索、交谈、练功、查看状态和战斗。

两者之间主要通过“江湖行动”和“剧情”按钮切换，没有充分共享任务、地点、敌人和后果。

直接结果是：

- 玩家从剧情地点离开后，可以前往与当前任务无关的区域；
- 返回剧情时又瞬间回到原事件，产生空间跳跃；
- 训练获得的熟练和招式未稳定改变剧情选项；
- 普通敌人与剧情威胁没有明确因果关系；
- 剧情像一条主线，MUD 像旁边临时搭建的训练场。

## 2.2 玩家没有持续可见的目标

当前界面没有始终回答以下问题：

```text
我是谁？
我在哪里？
我现在要做什么？
为什么要做？
如果拖延会发生什么？
```

玩家进入王允府外巷后，会同时看到历史背景、人物关系、剧情选项、江湖行动入口、状态入口和命令系统，却没有一个足够明确的任务焦点。

## 2.3 开场同时引入过多系统

玩家在短时间内接触：

- 历史卷首；
- 人物序章；
- 渐显正文；
- 剧情选项；
- 自由探索；
- 命令输入；
- 状态、行囊、武学和记录；
- 练功和熟练度；
- 等级与阅历；
- 战斗的气力、架势和敌方意图。

单个系统并不复杂，但同时出现会造成显著认知负担。

## 2.4 状态概念过多且缺少解释

现有概念包括：

- 等级；
- 阅历；
- 气血；
- 体力；
- 内息；
- 伤势；
- 风声；
- 余时；
- 局势压力；
- 城门进度；
- 熟练度；
- 人情。

主要问题：

1. `余时`、`局势压力`和`城门进度`承担相近的时间压力；
2. 玩家只看到名称和数值，看不到当前后果；
3. 行动前无法判断状态变化会关闭什么机会；
4. 状态第一次出现时没有情境化教学；
5. 主界面与详情页没有明确的信息层级。

## 2.5 信息时间顺序不统一

当前不同区域采用不同顺序：

- 剧情正文从上往下；
- 行动结果使用覆盖式弹层；
- 江湖日志旧信息在上、新信息在下；
- 记录页最新信息在上；
- 战斗状态、敌方意图和行动结果混排。

玩家无法稳定判断屏幕上哪一条是最新发生的事情。

## 2.6 自由行动缺少叙事动机

“搜索、训练、调息、迎战”虽然会改变数值，但缺少清晰的任务关系：

- 搜索什么，为什么值得搜索；
- 练功是为了应对哪个已知障碍；
- 击败劫掠者会帮助哪个人物；
- 获得招式后能在哪个剧情节点使用；
- 不采取行动会失去什么。

机制行为必须服务于玩家已经理解的目标，否则只会成为额外操作。

## 2.7 地点开放范围破坏剧情因果

玩家在剧情早期即可移动到较远地点，容易提前看见尚未发生的场景、人物或敌人。地点图虽然存在，但没有按照历史阶段和任务进度逐步开放。

## 2.8 文字命令过早成为视觉重点

命令输入是为熟悉 MUD 的玩家提供效率，不应成为新玩家理解游戏的前提。目前命令栏与主要按钮同时出现，容易让玩家误以为必须学习命令。

# 3. 重构核心原则

## 3.1 只保留一个游戏世界

取消“剧情模式”和“江湖行动模式”的并列关系。

统一后的结构：

```text
当前地点
+ 当前任务
+ 稳定环境描述
+ 按时间排列的行动记录
+ 当前可用行动
```

剧情不再通过单独模式播放，而是在以下条件满足时自然触发：

- 到达地点；
- 与特定 NPC 交谈；
- 搜索到物品；
- 时间或城门进度达到阶段；
- 战斗结束；
- 玩家作出不可逆承诺。

## 3.2 始终解释玩家为什么行动

每个场景固定展示：

- **当前目标**：需要完成什么；
- **推荐地点**：下一步通常去哪里；
- **时间压力**：继续拖延会发生什么；
- **可选准备**：哪些行动能提高成功机会。

## 3.3 机制结果必须改变剧情

探索、训练和战斗获得的每一项重要成果，至少改变以下一项：

- 新剧情选项；
- 选项代价；
- 战斗动作；
- NPC 态度；
- 人物命运；
- 物品或情报；
- 下一地点的状态。

纯数值成长不得成为主要奖励。

## 3.4 逐步开放复杂度

新玩家不应在第一分钟看到全部系统。功能按照实际需要依次开放，并提供一次性解释。

## 3.5 所有实时信息使用同一时间方向

统一规则：

```text
旧信息在上
最新信息在下
当前操作始终在最下方
```

# 4. 目标玩家流程

以杜缄开局为例：

```text
卷首背景
→ 人物身份
→ 王允府外巷
→ 明确任务：护送许简前往城西旧宅
→ 观察换岗武吏
→ 与赵衡交谈并接令
→ 检查短剑和止血散
→ 前往西市废仓
→ 自动触发伏击
→ 完成第一场多回合战斗
→ 战斗结果直接改变许简伤势、敌人去向和可用线索
→ 选择救人、追击或处理火油
→ 继续探索和剧情
```

玩家不再需要主动判断“现在该去剧情还是江湖行动”。所有事情都发生在同一个地点流程中。

# 5. 统一主界面

## 5.1 信息架构

```text
章节与城门进度
当前任务
当前地点
稳定场景描述
────────────
按时间排序的行动记录
────────────
当前可用行动
命令输入（默认折叠）
```

## 5.2 当前任务卡

任务卡始终显示：

```text
当前目标：护送许简前往城西旧宅

建议下一步：与赵衡交谈，取得文书
时间压力：城门正在收紧
可选准备：检查装备、观察守卫
```

重大分歧后任务卡必须更新。例如：

```text
当前目标：决定先救许简还是追持卷者
```

## 5.3 行动分组

行动按照意义分组，不再混成一个按钮列表。

### 主线行动

- 推进当前任务；
- 使用金色或印章式强调；
- 同一时间通常只有 1～2 项。

### 地点行动

- 观察；
- 搜索；
- 交谈；
- 使用环境。

### 准备行动

- 检查装备；
- 练习与当前障碍相关的招式；
- 调息或治疗。

### 移动行动

只显示当前阶段合理且相邻的地点。

## 5.4 命令输入

命令栏默认折叠在“更多操作”中。

- 按钮覆盖全部核心功能；
- 命令不提供隐藏优势；
- 玩家主动展开后显示示例；
- 输入错误不消耗资源；
- 命令和按钮调用同一规则入口。

# 6. 状态体系精简

## 6.1 主界面状态

主界面只显示四项：

| 状态 | 含义 | 直接用途 |
|---|---|---|
| 气血 | 当前战斗承受能力 | 归零后战败并增加伤势 |
| 体力 | 普通行动资源 | 移动、训练、攻击和格挡 |
| 内息 | 武学资源 | 施展进阶招式 |
| 城门进度 | 历史局势时钟 | 关闭地点、人物和机会 |

## 6.2 二级状态

放入人物详情：

| 状态 | 含义 |
|---|---|
| 伤势 | 跨战斗持续的身体损伤，限制强攻和轻身 |
| 风声 | 外界对玩家身份与行动的关注程度 |
| 熟练度 | 当前兵器掌握程度和招式解锁进度 |
| 阅历 | 本局实战成长进度 |
| 人情 | 可请求帮助的有限关系资源 |
| 人物关系 | NPC 对玩家的信任、戒备和债务 |

## 6.3 合并状态

- 删除独立 `局势压力`；
- 删除独立 `余时`展示；
- 两者统一映射为“城门进度”；
- 等级不放在主界面，改称“实战层次”，仅在人物或武学页显示。

## 6.4 城门进度

不显示抽象数字，使用阶段：

| 阶段 | 玩家可见影响 |
|---|---|
| 城中尚稳 | 地点正常开放，可以进行有限准备 |
| 盘查加严 | 部分身份行动风险上升 |
| 闭门鼓急 | 训练机会关闭，军属向城门聚集 |
| 城门将闭 | 支线地点关闭，必须处理主要目标 |
| 长安失守 | 进入终局和余波 |

# 7. 状态解释机制

## 7.1 状态不能只显示名称

错误示例：

```text
风声：暴露
```

正确示例：

```text
风声：暴露

你的身份已被部分武吏确认。
· 普通盘查更难蒙混
· 部分伪装行动关闭
· 再次公开动武将进入“缉拿”
```

## 7.2 行动前预告

每个重要行动显示可预测影响：

```text
【亮出剑印喝止追兵】

可能后果：
风声：起疑 → 暴露
王允府身份将被武吏认出
```

战斗动作示例：

```text
【横剑截流】

消耗：内息 1
效果：保护身后目标，削弱敌方架势
使用后仍有 1 点内息
```

## 7.3 行动后解释

行动结果附带变化原因：

```text
风声：起疑 → 暴露
原因：你在三名武吏面前公开展示了师门剑印
```

避免只显示 `风声 +1`。

## 7.4 第一次出现时教学

状态第一次变化时，显示一次简短说明：

> **风声升至“起疑”**  
> 风声表示外界对你身份和行动的关注。风声越高，盘查、追兵和可用路线都会变化。

教学只自动出现一次，之后可点击状态名称重看。

## 7.5 状态规则页

状态详情需要展示：

- 当前阶段；
- 当前实际影响；
- 下一阶段影响；
- 哪些行为会恶化；
- 哪些行为可以缓解；
- 与哪些人物、地点和行动相关。

## 7.6 锁定行动解释

锁定选项必须说明全部关键原因：

```text
【踏墙换位】暂不可用

· 当前穿着札甲
· 需要至少 1 点内息
```

# 8. 信息时间线统一

## 8.1 固定顺序

所有实时信息统一为：

```text
旧信息在上
新信息在下
最新行动最接近操作区
```

适用范围：

- 剧情行动；
- 探索结果；
- NPC 对话；
- 训练反馈；
- 战斗回合；
- 状态变化；
- 单局行动记录。

通关历史列表可以保留最新周目在上，但单个周目内部必须从旧到新。

## 8.2 场景描述与行动记录分离

### 稳定场景描述

只写不会因单次行动立即改变的环境：

> 西市废仓门轴断裂，仓内堆着麻包和火油罐。

### 行动记录

写本局刚刚发生的事情：

> 你检查了仓门后的脚印。  
> 发现：有人拖着伤腿向军属里巷移动。  
> 获得：魏石去向线索。

## 8.3 取消打断式结果弹窗

移除“此念既出·后果已生”覆盖层。行动结果直接追加到时间线，然后在底部开放下一批行动。

结果格式：

```text
【你选择：留下替许简止血】

你按住他肋下的伤口，血终于不再继续涌出。
许简暂时脱离危险。

余时变化改为：城门进度向前推进
许简：重伤 → 稳定
```

## 8.4 自动滚动

- 新结果产生时滚动到最新消息；
- 玩家主动向上查看时暂停自动滚动；
- 新内容到达后显示“有新消息”；
- 点击提示回到底部；
- 返回场景时不重新播放已经读过的文字。

## 8.5 文字渐显边界

- 初次出现的卷首、人物序章和重要场景可以按语句渐显；
- 玩家可以随时“显示全文”；
- 开启系统“减少动态效果”时直接显示；
- 普通行动日志不逐字播放，只对最新条目做短暂淡入；
- 已经读过的场景不得因查看状态、行囊或返回地点而重新渐显；
- 渐显期间应保留明确的跳过入口，不能让玩家误以为页面卡死。

# 9. 战斗信息重构

## 9.1 固定战斗区

战斗顶部固定显示：

- 敌方名称；
- 气力；
- 架势；
- 距离；
- 下一步公开意图；
- 保护目标和环境危险。

## 9.2 回合卡片

一轮行动完整显示在同一张卡片：

```text
第三轮

敌方意图：蓄力重击

你：横剑截流，削弱架势 2
敌：长刀擦过肩甲，造成气血伤害 4

状态变化：
内息 2 → 1
敌方架势 3 → 1
```

最新回合使用淡金高亮，旧回合逐渐降低视觉权重。

## 9.3 第一场战斗教学

第一次伏击按照以下顺序教学：

1. 先解释敌方意图；
2. 只开放“试探”和“格挡”；
3. 展示架势变化；
4. 再开放快攻；
5. 达到条件后开放第一个招式；
6. 战后明确说明结果如何改变许简和持卷者命运。

## 9.4 战斗与剧情绑定

西市伏击必须直接替换原剧情一次性战斗选项。

战斗结果至少影响：

- 许简是否受伤；
- 持卷者是否逃脱；
- 玩家熟练度；
- 风声；
- 火油是否成为后续危险；
- 能否追踪任朔路线痕迹。

# 10. 探索、训练与成长整合

## 10.1 探索必须有当前理由

搜索按钮需要说明目标：

```text
【检查仓门脚印】
目的：寻找袭击者撤退方向
```

避免没有上下文的通用“搜索”。

## 10.2 训练必须对应已知障碍

在玩家知道“西市巷口狭窄、可能遭伏击”后，才开放：

```text
【在府墙间练习窄处进退】
可能解锁：守剑领悟
代价：推进城门进度
```

训练不是独立小游戏，而是为当前任务准备。

## 10.3 敌人必须与地点和任务有关

敌人不以“经验来源”出现，而以世界角色出现：

- 追捕武吏；
- 趁乱劫掠者；
- 溃兵；
- 魏石部下；
- 董卓旧部联络者；
- 可切磋的同门和老卒。

击败后不会刷新，并改变地点状态。

## 10.4 成长必须改变动作

熟练和阅历主要用于：

- 解锁截腕、横剑截流、踏墙换位等动词；
- 降低某类行动代价；
- 看出敌人意图；
- 使用环境；
- 在剧情中提供新方法。

不以单纯攻击数值增长作为主要反馈。

# 17. 验收标准

## 17.1 玩家理解

试玩者在前 3 分钟内能够回答：

- 我扮演谁；
- 我当前在哪里；
- 当前目标是什么；
- 下一步可以去哪里；
- 城门进度意味着什么。

## 17.2 机制理解

第一次战斗后能够回答：

- 气血、体力和内息的区别；
- 敌方架势有什么作用；
- 风声升高会造成什么；
- 新招式如何改变行动。

## 17.3 流程一致性

- 不存在“返回剧情”造成的地点跳跃；
- 最新信息始终位于时间线底部；
- 返回场景不重新播放旧文本；
- 训练和战斗成果至少改变一个剧情条件；
- 地点不会提前出现不合时序的人物和事件。

## 17.4 可用性

- 不输入命令也能完成全部流程；
- 命令栏默认折叠；
- 锁定行动说明原因；
- 所有玩家状态均有简短解释和完整规则页；
- 主界面同时可见状态不超过四项；
- 所有核心触控目标不低于 44 像素。

## 17.5 E2E 测试

至少覆盖：

- 新玩家引导；
- 当前任务更新；
- 地点移动触发剧情；
- 状态首次变化教学；
- 城门进度关闭训练；
- 第一场多回合战斗；
- 战斗结果改变剧情；
- 时间线从旧到新；
- 向上翻阅时的新消息提示；
- 存档恢复至同一地点和任务；
- 杜缄完整路线；
- 任朔代表路线；
- 移动端和桌面响应式；
- 浏览器控制台、请求和页面脚本无错误。

# 19. 当前原型的处理结论

当前 MUD v0.1 不作为最终界面继续扩展。

应当：

- 保留底层探索、训练、战斗、命令和存档代码作为验证资产；
- 停止在独立 MUD 页面继续增加地点和敌人；
- 优先完成统一界面和杜缄开局垂直切片；
- 在统一切片通过玩家测试后，再迁移剩余内容。

最终目标不是“剧情游戏加一个 MUD 页面”，而是：

> 玩家在同一个长安世界里观察、交谈、移动、练功、战斗和作出历史抉择；剧情是世界发生的事情，不是需要切换进入的另一个模式。
