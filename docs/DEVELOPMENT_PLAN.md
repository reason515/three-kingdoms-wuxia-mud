# 1. 计划目标与边界

## 1.1 MVP 交付目标

在第 10 个工作周结束时，交付可由 20～50 名内测玩家访问的 PWA 版本。玩家应能在长安完成以下闭环：

```text
创建角色 → 客店降生 → 探索房间 → 交谈/接任务 → 巡逻战斗
→ 修炼或离线挂机 → 上线收取收益 → 升级/学习技能 → 继续探索
```

MVP 内容边界遵循 `RECONSTRUCTION.md`：长安 20～30 间房、两条野外路径、1～10 级、10～15 个技能、15～20 个任务、4 种挂机模式。

## 1.2 不纳入本计划

以下内容不阻塞 MVP，全部进入 MVP 后 Backlog：

- PVP、交易行、排行榜、帮会、师徒、坐骑与宠物；
- 跨州地图、完整天气系统、锻造/装备品质循环；
- 正式可视化内容编辑器；MVP 只提供 YAML 校验、预览与调试工具；
- Redis、PostgreSQL、微服务拆分；MVP 使用 SQLite + 单体 Node.js 服务。

## 1.3 开发节奏

- 以**工作周**计，不绑定自然日；每周末必须形成可运行、可演示的版本。
- 直接提交 `main`，但每次提交前执行适用的质量门禁。
- 不以“模块写完”为完成标准，而以“玩家可从 UI 走通一个行为”作为完成标准。
- 每个玩家可感知的功能改动必须同步新增或更新**对应领域 E2E**；每次改动均运行烟雾 E2E。

# 6. 每类改动的执行模板

## 6.1 引擎逻辑改动

示例：增加“破甲”伤害效果。

```text
需求：R04 战斗中技能可改变防御结算。
代码：packages/server/src/engine/combat.ts
单元：combat.test.ts 覆盖轻甲/重甲/无甲及边界数值。
集成：战斗胜利后经验与熟练度正确写入数据库。
E2E：combat.spec.ts 验证玩家使用破甲招式后战报与敌人状态变化。
必跑：pnpm test:e2e -- combat；pnpm test:e2e（Smoke）。
提交：combat: 实现西凉刀法破甲效果
```

## 6.2 内容改动

示例：新增长安西市房间。

```text
需求：R03 玩家可探索新的西市区域。
内容：packages/content/rooms/changan/west_market.yml
校验：schema、房间 ID 唯一、出口存在且双向连通、NPC/怪物引用存在。
E2E：room.spec.ts 验证从相邻房间可进入、房间描述/NPC/出口正确展示。
必跑：pnpm validate；pnpm test:e2e -- room；pnpm test:e2e（Smoke）。
提交：content: 新增长安西市房间
```

## 6.3 纯重构或文档改动

- **纯重构**：不得改变协议或玩家行为；添加/更新单元和集成测试证明行为不变，运行 Smoke；提交使用对应 scope，如 `server: 拆分战斗结算模块`。
- **纯文档**：只需链接/格式检查；提交 `docs: 更新MVP开发计划`。若文档改动同步修改了可执行脚本或配置，则按 `chore` 执行 Smoke。

# 7. 测试与环境实施顺序

开发工作流本身也是 Phase 0 的交付物，应按以下顺序落实：

1. **先建立隔离数据库**：每个 Vitest/Playwright 测试运行使用独立 SQLite 文件，测试后删除；不得共享开发库。
2. **再建立确定性时钟**：server 导出 `Clock` 接口；单元/集成测试用 FakeClock 推进 tick，禁止依赖真实 `setTimeout`。
3. **再建立 E2E 调试入口**：仅在 `NODE_ENV=test` 启用，用于创建测试账号、固定随机种子、推进 tick、预置房间怪物；生产环境必须禁用。
4. **再建立 Smoke**：按 W2～W4 逐步实现 8 条，不允许长期 skip。
5. **最后接 Husky 与 CI**：避免在未稳定前让 pre-commit 频繁阻塞开发；从 W3 起启用 typecheck/lint/validate，从 W4 起启用 Smoke。

# 8. 风险控制与调整规则

| 风险 | 触发信号 | 处理规则 |
|---|---|---|
| Smoke 超过 45 秒 | 连续 3 次 > 45 秒 | 检查是否错误依赖真实 tick；用 test clock 或并行 fixture 优化 |
| E2E 脆弱 | 同一 commit 连续重跑出现非确定性失败 | 先修测试/等待条件，不允许用 retry 掩盖真实问题 |
| 内容生产拖慢引擎 | W6 时 M1 Gate 未稳定 | 停止添加内容，优先修复引擎与测试基座 |
| 管理后台吞噬工期 | W7 前未完成核心内容 60% | 降级为只读内容预览 + CLI 校验，编辑功能延后 |
| 离线挂机数值失衡 | 内部模拟显示挂机收益高于在线探索 | 先调收益/上限，不通过新增内容掩盖数值问题 |
| 单人开发疲劳 | 单周未形成可演示纵切 | 下一周缩小范围，只保留一个玩家闭环，删除非阻塞任务 |
