# 1. 一句话定位

一款移动端优先、选择驱动、可反复挑战的三国武侠文字 RPG。玩家从无名之人出发，在真实历史事件的夹缝中调查隐秘、结交人物、修习武艺，并以不可兼得的选择决定局部命运，逐步拼合被史书掩埋的真相。

# 2. 核心体验

游戏希望玩家反复产生以下体验：

- 我知道历史大势将走向何方，却不知道它背后究竟发生过什么；
- 我无法救下所有人，也无法同时守住任务、信义、名声和性命；
- 我选择的能力和立场真正改变了可用行动；
- 我这一局看到的只是某个身份下的局部真相；
- 我想换一种身份、路线或方法，再进入这段历史一次。

核心理念：

> **大势或不可改，人心自有去处。**

# 3. 题材与叙事方向

## 3.1 历史骨架

游戏以三国史实为骨架。董卓之死、长安陷落、官渡胜负、赤壁之战、荆州易主、诸葛亮北伐等重大事件应保持历史辨识度。

玩家不应轻易通过一次选择彻底改写天下格局，否则历史将失去重量。

## 3.2 历史夹缝

玩家主要介入史书没有详细记载的部分：

- 情报为何迟到或被篡改；
- 某人为何改变立场；
- 一份军令或名册落入谁手；
- 无名者如何在战争中生存；
- 某位历史人物获得了什么错误信息；
- 后世流传的忠奸评价是否符合事实。

这些内容允许适度虚构，但虚构必须服务于时代逻辑、人物动机和玩家决策。

## 3.3 江湖化表达

历史不从帝王将相的全知视角展开，而通过驿站、客栈、军营、渡口、废寺、商队、门派、密信、暗号和江湖传闻被玩家亲历。

历史人物可以是庇护者、上司、敌手、传闻来源或少数关键会面对象，但不能挤压玩家和原创人物的成长空间，也不应被统一塑造成武林高手。

## 3.4 叙事参考的转化原则

叙事气质可以吸收以下类型的长处，但不复制具体人物、组织、情节和标志性设定：

- 情报叙事：多方渗透、真假信息、身份怀疑和小人物对大局的局部影响；
- 江湖史诗：底层人物进入时代洪流，武学成长与人物成长同步；
- 历史人物弧光：表现人物为何作出历史选择，以及理想与现实之间的代价。

# 4. 核心玩法循环

```text
选择身份与初始专长
→ 进入历史篇章
→ 阅读场景和局势
→ 从2～5个行动中取舍
→ 改变资源、关系、疑迹和情报
→ 进入条件事件、交锋或历史节点
→ 形成结局与人物命运
→ 获得史证、称号和挑战记录
→ 以新身份或新构筑再次进入
```

每个历史篇章建议：

- 单局时长 15～30 分钟；
- 12～20 次有效选择；
- 2～3 个阶段性历史节点；
- 6 个以上主结局；
- 多组人物命运和江湖评语；
- 在有限内容内支持多路线重玩。

# 5. 选择设计

## 5.1 选择必须改变状态

每个选择至少改变以下一项：

- 能力或资源；
- 人物关系；
- 阵营立场；
- 身份暴露；
- 情报认知；
- 后续路线；
- 延迟后果。

不保留只产生不同回复、但不改变状态或认知的伪选择。

## 5.2 选择必须有机会成本

主要稀缺资源为：

- **时间**：无法同时完成全部目标；
- **伤势**：强行动会影响后续能力；
- **疑迹**：决定敌方警觉和身份暴露；
- **人情**：关键帮助不可无限调用；
- **情报**：可用于判断和交易，但可能失真。

选项的理想结构不是“正确与错误”，而是“得到什么、放弃什么、愿意承担什么风险”。

## 5.3 信息不完整但必须公平

系统不公开所有后果，但应通过场景线索、人物反应和风险标签提供合理判断依据。失败应来自玩家接受了风险或误判了信息，而不是作者毫无提示地惩罚玩家。

## 5.4 决策密度

不是每次点击都必须决定生死，但每次选择都应改变局面。重大、不可逆的抉择建议每 3～5 次行动出现一次，避免持续高压导致决策疲劳。

# 6. 角色成长

## 6.1 核心能力

第一版建议保留四项能力：

- **武**：战斗、威慑、护送和正面突破；
- **谋**：调查、识破、布局和判断；
- **义**：人物信任、守诺和侠义路线；
- **势**：身份、声望、人脉和阵营影响。

## 6.2 专长

专长直接解锁行动方式：

- 剑术：决斗、保护、破阵；
- 轻功：潜入、追踪、撤离；
- 医术：救治、验伤、辨毒；
- 辩才：说服、欺骗、离间；
- 谍术：暗号、伪装、反侦察；
- 兵法：判断军势、路线和历史节点。

成长奖励应优先提供新选择，而不是简单的“数值 +5”。

## 6.3 局外解锁

局外成长用于扩大可能性：

- 新身份；
- 新专长和武学；
- 新事件和人物路线；
- 新的初始信物；
- 已确认或相互矛盾的史证；
- 更高难度与特殊挑战。

永久成长不应让旧内容失去挑战。

# 7. 多周目与真相系统

## 7.1 史证

玩家在不同路线中获得：

- 密信残页；
- 军令和档案；
- 人物证词；
- 江湖传闻；
- 被涂改的名册；
- 临终遗言；
- 可疑物证。

史证可能互相支持，也可能互相矛盾。游戏应区分“玩家见过的说法”和“已经确认的事实”。

## 7.2 江湖史卷

局外以“江湖史卷”记录：

- 历史篇章完成度；
- 已发现史证；
- 已知人物身份；
- 人物命运分支；
- 主结局和特殊称号；
- 尚未解释的疑点。

部分完整真相只能通过多个身份和互斥路线拼合。

## 7.3 执笔定史

在掌握足够证据后，玩家可以决定如何记录某一事件。该选择不一定改变已经发生的历史，却会决定谁被记住、谁背负污名，以及玩家自己的史家立场。

# 8. 人物与结局

## 8.1 人物设计

重要人物应具备：

- 公开目标；
- 私人牵挂；
- 阵营或政治立场；
- 不愿公开的秘密；
- 在历史压力下必须支付的代价。

原创人物承担玩家能够深度介入的命运线，历史人物提供时代压力、政治逻辑和少量关键交汇。

## 8.2 结局结构

结算由三层组成：

1. **主角结局**：玩家最终成为何种人；
2. **命运图谱**：关键人物的生死、归属、关系和名声；
3. **江湖评语**：根据行为生成简短人物小传。

结局不只区分好坏，更应体现玩家选择的价值与代价。

# 9. 战斗与文字表现

## 9.1 战斗原则

战斗采用 3～6 轮情境化选择。每轮选择应与环境、任务和人物目标相关，例如抢攻、守势、试探、借地形、保护同伴、欺骗、撤离。

胜负之外还应存在：

- 是否负伤；
- 是否暴露身份；
- 是否保护目标；
- 是否获得证据；
- 是否错过时间；
- 是否与对手结下后续关系。

## 9.2 文字结构

场景与战斗文本遵循：

1. 以一两个具体细节建立画面；
2. 用清晰动词表现动作和距离；
3. 让环境参与冲突；
4. 明确表现局势变化；
5. 以简短系统提示补充必要状态。

避免长篇堆砌招式名称，也避免只显示“造成 12 点伤害”。

# 10. 重复挑战与排行榜

## 10.1 重玩动力优先级

1. 补全真相；
2. 改变人物命运；
3. 尝试不同身份和能力构筑；
4. 收集结局、称号和江湖评语；
5. 完成高难历史挑战；
6. 参与排行榜竞争。

排行榜是熟悉内容后的高阶目标，不承担最初的重玩动力。

## 10.2 历史挑战

挑战条件可以包括：

- 不杀一人；
- 不向任何阵营效忠；
- 全程保持真实身份；
- 在重伤状态下护送目标；
- 只携带一件物品；
- 保住全部关键证人；
- 在低疑迹下取得核心证据。

## 10.3 排行榜

采用统一篇章、身份、难度和事件种子，按不同目标设置分榜：

- 综合表现；
- 无伤；
- 低疑迹；
- 救援；
- 情报完整度；
- 极限难度；
- 特定身份。

# 11. 移动端交互

- 当前场景提供 2～5 个上下文选项；
- 主要流程不调用键盘；
- 场景文本控制在适合手机阅读的长度；
- 底部固定行动区；
- 人物、武学、行囊、关系和史卷通过单层抽屉查看；
- 选项优先显示行动意图、必要条件和主要风险；
- 自然语言输入仅作为后续可选增强，不影响核心流程与排行公平。

# 12. 首个垂直切片：《长安残照》

## 12.1 历史锚点

- 时间：初平三年（192 年）；
- 董卓已死，王允掌权；
- 吕布与并州军处境不稳；
- 李傕、郭汜即将反攻长安；
- 最终长安陷落、王允身死、吕布败走。

## 12.2 玩家介入空间

- 一份涉及多方人物的名册落入谁手；
- 某位密使是否完成任务；
- 一群百姓能否逃出长安；
- 一封密信是否被篡改；
- 谁被当作叛徒写入记录；
- 玩家最终忠于任务、人物、阵营还是自己。

## 12.3 MVP 范围

- 3 种初始身份；
- 4 项核心能力；
- 6 种专长中的部分组合；
- 5～7 名重要人物；
- 约 30 个条件事件；
- 6 个以上主结局；
- 人物命运图谱；
- 基础江湖史卷；
- 本地挑战记录，正式排行榜后置。

## 12.4 验证问题

1. 玩家是否愿意通关后立即换一种身份再试？
2. 玩家能否说清楚本局放弃了什么、为什么放弃？
3. 玩家是否相信另一条路线能揭示新的事实，而不只是不同文本？
4. 玩家是否会为某个人物命运主动改变策略？
5. 文字战斗是否在不依赖画面的情况下清晰、紧张且有空间感？

# 13. 当前不做

- 真正多人在线；
- 大型开放世界；
- 完全自由的 AI 剧情生成；
- 复杂经济和装备系统；
- 以挂机练功为核心的成长；
- 无限制永久属性提升；
- 正式赛季与付费系统。
