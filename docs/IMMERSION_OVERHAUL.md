# 1. 整改目标

本轮不以增加分支数量为目标，而解决四个直接影响代入感的问题：

1. 玩家选择后只能看到下一场景，看不到行动如何发生；
2. 尾声无条件提及可能已经被捕、死亡或失散的人物；
3. 选项提示直接暴露系统收益，玩家容易按数值而非角色判断；
4. 赵衡、魏石与普通军属的关系主要依赖背景说明，缺乏玩家亲历的情感依据。

整改后的核心循环为：

```text
阅读具体局势
→ 依据角色能够知道的风险行动
→ 观看动作、人物反应和代价
→ 世界状态改变
→ 下一场景承认刚才发生的事
→ 条件化尾声回收人物、物件与历史余波
```

# 2. 已完成改造

## 2.1 即时行动回响

内容 Schema 的每条 `resolution` 新增可选 `resultTextKey`。两份真实内容包目前共包含：

- 杜缄：101 条结果文本；
- 任朔：58 条结果文本；
- 合计：159 条按具体结果分辨的行动回响。

Web 原型点击选项后先显示“此念既出·后果已生”结果页，再进入下一事件。场景铺陈与人物序章采用按语句渐显，而不是一次展示整段：

- 每句以淡墨渐清的方式依次出现；
- 正文显现完成前暂不展示行动选项；
- 玩家可随时选择“显示全文”；
- 系统启用“减少动态效果”时直接显示全文；
- 重玩和自动化测试可以立即跳过，不强制等待。

结果文本优先说明：

- 身体动作和空间结果；
- NPC 当场反应；
- 时间、伤势、暴露或物品代价如何在世界中发生；
- 该选择留下的情感余波。

## 2.2 条件场景与尾声

事件新增有序 `textVariants`，结局新增 `summaryVariants`。

杜缄 D12 会区分：

- 收押；
- 阿萝与许简均在；
- 只有阿萝一行；
- 只有许简与记录；
- 无关键人物同行的基础状态。

任朔 R10 会区分：

- 被捕或收押；
- 军属队伍生还；
- 只有阿萝与阿禾；
- 独自离开。

基础文本不得无条件写入命运不确定的人物。

## 2.3 选择提示去系统化

按钮副文案只显示角色能够预判的风险，例如：

- 会耽搁行程；
- 需要动用内息；
- 身份可能暴露；
- 会伤及彼此信任；
- 可能有人被留下；
- 会改变物件去向。

不再直接展示“进入某路线”“获得领悟”“触发某结局”等作者或系统语言。

## 2.4 人物关系落地

### 赵衡

- 开局增加雪夜替杜缄养母送药的共同经历；
- 杜缄有充分理由先信任赵衡，而非开场便知道他有罪；
- 左肩旧伤从首周目公共描写中移除，只在任朔周目见闻存在时出现；
- 决裂由“识破坏人”改为“承认有恩之人正在作恶”。

### 魏石

- R02 的双环路标实际替任朔避开塌梁；
- 魏石的旧恩在本局产生真实帮助，不再只是两年前挡箭的说明；
- 他的救军属与趁乱自立同时成立，避免单一反派化。

### 阿禾

阿萝弟弟正式命名为阿禾，作为普通人命运的情感锚点：

- 木剑在初见、过门和尾声反复出现；
- 与阿萝同步记录同行、失散、被捕和安全状态；
- 不同主角会让他学到不同的持剑或持刀姿势；
- 结局通过木剑的去向回收玩家对“侠”的理解。

## 2.5 双路线身份差异

- 杜缄线继续突出文书、窄巷、短剑、同门和受托压力；
- 任朔 R06 强调甲具在撤退队列中的分配，而非单纯个人换甲；
- 任朔 R07 改为伤兵歇脚处的撤阵口授，以军中伤亡记录形成护军或破阵理解；
- 任朔的行动反馈更多表现队形、旧伤、军哨、马匹和军法。

## 2.6 大历史卷首与信息着色

首次进入篇章时增加共享卷首《长安将倾》，使用小人物能够感知的城门、军报、流言和籍贯盘查，依次解释：

- 董卓被诛以及王允、吕布在事件中的位置；
- 李傕、郭汜重新收拢董卓旧部；
- 赦免消息不明如何放大军民恐慌；
- 一张籍贯名册为何会从行政文书变成生死依据；
- 历史固定结局与无名人物可改变命运之间的边界。

卷首只在首次进入时自动出现，之后可从标题页的“卷首·长安将倾”主动重读。

正文关键实体采用克制的古籍色彩标识：

| 类型 | 表现 |
|---|---|
| 人物与历史人物 | 暗朱 |
| 兵器、信物、文书与秘籍 | 鎏金 |
| 地点 | 青黛 |
| 武学招式 | 青绿 |
| 阵营与历史组织 | 灰紫 |

标识由内容包中的人物、物品、地点和武学名称自动生成，并在卷首、场景正文、行动结果与结局摘要中统一生效。

# 4. 质量门禁

`tools/validate_immersion.py` 自动检查：

- 每条结果是否存在有意义的反馈文本；
- 选项提示是否残留路线、结局或数值化系统语言；
- 阿禾命运是否与阿萝保持连续；
- 所有尾声事件是否具备条件文本；
- 所有结局是否具备生死或传承摘要变体；
- 内容版本是否已经升级至 v0.2.0。

该检查已经加入 `tools/validate_all.py`。

# 6. 下一轮人工测试重点

自动化只能证明状态连续，不能替代玩家感受。纸面或 Web 测试应重点记录：

1. 玩家是否会在结果页停留阅读；
2. 玩家第一次对赵衡或魏石产生怀疑发生在哪个节点；
3. 玩家是否能说出阿禾是谁以及木剑最后如何处理；
4. 玩家是否因为人物关系而选择了非最优资源路线；
5. 第二人物视角是否像另一种身份，而非同一流程换皮；
6. 玩家是否能用自己的话复述一次“行动—代价—后果”；
7. 尾声是否出现任何人物、伤势、兵器或自由状态穿帮。
