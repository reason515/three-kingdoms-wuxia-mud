---

> 本规范从游戏概念图（`docs/game-concept.html`）提炼而来，作为后续所有界面的统一视觉与交互依据。
>
> 设计基调：**水墨宣纸 + 描金朱印 + 衬线古风**。整体气质要“沉、雅、有江湖气”，避免现代扁平 SaaS 的轻浮感。
>
> 交互基调：**场景文本 + 2～5 个上下文行动选项**。移动端是主要体验，桌面端在此基础上扩展信息布局。

# 1. 设计原则

| 原则 | 说明 |
| --- | --- |
| **文字为主角** | UI 是文字的容器与陪衬，排版与留白服务于阅读，不喧宾夺主。 |
| **古而不旧** | 用古风配色与衬线字体营造年代感，但布局、响应式、交互遵循现代可用性标准。 |
| **沉稳克制** | 主色低饱和（墨、宣纸），强调色（朱印红、描金）少量点睛，不大面积铺色。 |
| **层次分明** | 通过纸面明度、描边、阴影区分层级，而非依赖大量颜色。 |
| **叙事感反馈** | 状态、条件、风险和结果都用世界内语气表达，UI 文案保持武侠口吻。 |
| **选择即行动** | 选项必须清晰表达行动意图、条件与主要风险，不使用含糊的“继续”代替决策。 |
| **移动端优先** | 核心流程无需键盘，单手即可完成阅读、选择和状态查看。 |

---

# 2. 设计 Token

所有界面**必须**使用以下 Token，不允许散落硬编码色值。建议落地为 CSS 变量（见 `:root`）或设计系统常量。

## 2.1 色彩

### 主色 · 纸与墨

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--ink` | `#1c1a17` | 主文字色、深色块背景、主按钮底 |
| `--ink-soft` | `#3a352e` | 次级正文、说明文字 |
| `--paper` | `#ece3d2` | 页面基础背景 |
| `--paper-2` | `#e2d6bd` | 次级背景、标签底、表头 |
| `--paper-card` | `#f3ecdc` | 卡片/面板背景 |

### 强调色 · 朱印与描金

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--seal` | `#9e2b25` | 印章红，关键强调、危险/警示、主 CTA 备选 |
| `--seal-deep` | `#7c211c` | 朱红深色，标题强调、hover 态 |
| `--gold` | `#a8842f` | 描金，描边、分隔线、次级强调 |
| `--gold-bright` | `#cba65b` | 亮金，深底上的高亮文字/图标 |

### 功能色 · 青玉与中性

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--jade` | `#4a6c5d` | 青玉绿，成功/正向、序号徽标、引用强调 |
| `--line` | `#c4b594` | 描边、分隔线、卡片边框 |

### 语义色（在上述基础色上约定）

| 语义 | 取色 | 场景 |
| --- | --- | --- |
| 成功 / 增益 | `--jade` | 学会武功、任务完成、属性提升 |
| 警示 / 危险 | `--seal` | 战斗受创、危险选择、删除存档 |
| 强调 / 重点 | `--gold` `--gold-bright` | 可交互高亮、关键道具 |
| 中性信息 | `--ink-soft` | 普通提示、旁白 |

> **配色比例参考 60 / 30 / 10**：纸墨主色约 60%，次级纸面/中性约 30%，朱印+描金强调不超过 10%。

## 2.2 字体

```css
--font-serif: "Noto Serif SC", "Songti SC", "STSong", "SimSun", serif; /* 标题、正文、叙事 */
--font-mono:  "Cascadia Code", "Consolas", "Courier New", monospace;   /* 命令输入、日志、数值 */
```

- **正文与叙事**：一律使用衬线字体 `--font-serif`，强化古风与阅读感。
- **命令行 / 战斗日志 / 数值**：使用等宽字体 `--font-mono`，体现 MUD 终端感。
- 不引入无衬线作为正文，以免破坏整体调性。

## 2.3 字号与字重（Type Scale）

| Token | 字号 | 行高 | 用途 |
| --- | --- | --- | --- |
| `--fs-display` | 48–54px | 1.2 | 页面主标题（游戏名） |
| `--fs-h1` | 28px | 1.3 | 区块大标题 |
| `--fs-h2` | 22px | 1.35 | 卡片/分组标题 |
| `--fs-h3` | 16px | 1.4 | 小标题 |
| `--fs-body` | 15px | 1.7 | 正文、叙事文本 |
| `--fs-small` | 13px | 1.6 | 说明、标签、脚注 |

- 标题使用字间距 `letter-spacing: 2–12px`（越大的标题间距越宽），强化碑刻/匾额感。
- 正文行高保持 `1.7`，保证大段文字可读性。

## 2.4 间距（4px 基准网格）

```
--space-1: 4px    --space-2: 8px    --space-3: 12px
--space-4: 16px   --space-5: 24px   --space-6: 32px
--space-7: 48px   --space-8: 64px
```

- 卡片内边距：`--space-5`（24px）。
- 区块之间间距：`--space-6`（32px）。
- 元素小间距：`--space-2 / --space-3`。

## 2.5 圆角

| Token | 值 | 用途 |
| --- | --- | --- |
| `--radius-sm` | 4px | 标签、输入框、小按钮 |
| `--radius-md` | 6px | 卡片、面板（默认） |
| `--radius-lg` | 8px | 大容器、对话框 |
| `--radius-pill` | 999px | 胶囊按钮、循环节点、状态条 |
| `--radius-round` | 50% | 序号徽标、头像、图标圆底 |

> 整体保持**小圆角**（4–8px），不用大圆角，以维持古朴方正气质。

## 2.6 阴影与描边

```css
--shadow-sm: 0 4px 12px rgba(28,26,23,0.06);   /* 卡片 */
--shadow-md: 0 6px 20px rgba(28,26,23,0.10);   /* 浮起面板 */
--shadow-lg: 0 12px 40px rgba(28,26,23,0.18);  /* 弹窗、头部 */
--border-1:  1px solid var(--line);            /* 默认描边 */
--border-accent: 2px solid var(--gold);        /* 强调描边 */
```

- 卡片左侧可用 `4px` 描金竖条（`border-left`）做强调变体。
- 重要容器用 `--border-accent` 描金/朱红边。

# 3. 组件规范

## 3.1 行动选项

行动选项是核心交互组件，普通场景显示 2～3 项，复杂抉择显示 3～4 项，阶段终局最多 5 项。

| 类型 | 样式 | 用途 |
| --- | --- | --- |
| **主要行动 Primary** | `--ink` 底 + `--gold-bright` 字 | 当前最直接的行动，但不暗示其一定正确 |
| **危险行动 Seal** | `--seal` 描边或局部强调 | 可能导致受伤、暴露、决裂等重大风险 |
| **普通行动 Secondary** | `--paper-card` 底 + `--line` 描边 | 调查、交涉、移动和资源取舍 |
| **条件行动 Locked** | 降低不透明度，保留条件说明 | 展示其他身份、武学或专长能够采取的路线 |

每个选项按需包含：

- 简短行动标题，如“拆信验看”“护送伤者”；
- 一行意图或结果倾向；
- 必要条件，如“需要：医术”；
- 主要代价或风险，如“疑迹上升”；
- 不提前公开本不应被角色知道的精确后果。

桌面端 hover 可轻微抬升 `translateY(-2px)`；移动端以按压态和描边变化反馈。禁用态降低不透明度至 `0.45`，但条件文字必须保持可读。

## 3.2 卡片 / 面板

- 背景 `--paper-card`，边框 `--border-1`，圆角 `--radius-md`，阴影 `--shadow-sm`。
- 顶部可加 `linear-gradient` 高光，模拟纸面受光。
- 悬停可交互卡片：抬升 + `--shadow-md`。

## 3.3 状态面板（角色属性）

- 使用 `--font-mono` 显示数值，强化系统感。
- 属性条（气血/内力）：胶囊形底槽 `--paper-2`，填充色按语义——气血用 `--seal`，内力用 `--jade`。
- 标签用 `--ink-soft`，数值用 `--ink`。

## 3.4 可选自然语言输入

- 不出现在主流程的固定底栏中，置于帮助或闲谈入口内。
- 深色终端风：底 `#211f1b`，文字 `#e6dcc6`，内描边 `rgba(168,132,47,0.3)`。
- 字体使用 `--font-mono`，placeholder 可使用「想问些什么？」。
- 自然语言不能提供选项之外的规则优势，也不能影响排行榜公平。

## 3.5 对话 / 叙事框

- 深色底（`#211f1b`）+ 浅色衬线字，营造"夜话/听闻"氛围。
- 玩家发言用「」引号包裹，色 `--gold-bright`。
- 世界回应/旁白用 `--jade` 左竖线引用样式，斜体。

## 3.6 标签 / Tag

- 底 `--paper-2`，描边 `--line`，圆角 `--radius-sm`，字号 `--fs-small`。
- 用于地点要素、物品属性、阵营标记。

## 3.7 印章 / 徽标

- 朱红方印（`--seal`）用于品牌点缀、重要标识，可轻微旋转 `-4deg` 增加手作感。
- 圆形序号徽标用 `--ink` 底 + `--gold-bright` 字，或 `--jade` 底 + 浅字。

# 4. 布局与栅格

- 内容最大宽度：`1080px`，居中。
- 阅读区（长文本/叙事）建议正文宽度 `640–760px`，避免行过长。
- 栅格：12 列思路，常用 2 / 3 / 4 列卡片网格，列间距 `--space-4`。
- 区块标题统一格式：`圆形序号 + 标题文字 + 渐隐描金分隔线`。

---

# 5. 响应式断点

| 断点 | 宽度 | 策略 |
| --- | --- | --- |
| 桌面 | ≥ 1024px | 中央叙事与行动区，侧边展示人物、线索和关系摘要 |
| 平板 | 760–1023px | 主叙事区优先，次要状态收入单层侧栏或抽屉 |
| 移动 | < 760px | 单列场景文本，底部固定行动区，状态进入单层抽屉 |

移动端优先保证：

- 场景文本进入页面后立即可读；
- 2～5 个行动选项在拇指可达区域内；
- 关键风险和条件无需二次点击即可看到；
- 人物、武学、行囊、关系和史卷最多通过一层抽屉访问；
- 核心流程不因软键盘弹出而中断。

---

# 6. 交互与动效

- 动效**克制而有韵**：默认过渡 `0.2s ease`，悬停抬升幅度 `2–3px`。
- 场景切换/文字出现可用淡入（`fade-in`，200–400ms），模拟"翻页/展卷"。
- 避免弹跳、强缩放等卡通化动效，保持沉稳。
- 关键反馈（受创、得宝、升级）可用色彩闪烁或印章盖下动效点睛，但短促。

---

# 7. 文案语气

- UI 文案保持**武侠/古白话**口吻，但不晦涩。
- 选项使用明确动词，避免只有“继续”“再看看”“或许如此”等无法判断意图的表述。
- 锁定或不可执行反馈要说明“缺少什么条件”或“当前为何不能这样做”。
  - ✅「你不通医理，无法判断伤口是否淬毒。」
  - ❌「条件不足」「Error: invalid action」
- 风险提示可以直白，但结果文本应保持叙事感，例如「疑迹上升」「与吕校尉的关系转冷」。
- 预留国际化：所有文案走文本 key 结构，便于后续英文版。

---

# 8. 落地建议

1. 将第 2 节 Token 落为 `:root` CSS 变量（或框架主题文件），全项目引用，禁止硬编码色值。
2. 优先完成移动端核心组件：场景文本、行动选项、状态变化、底部行动区和单层信息抽屉。
3. 再扩展桌面侧栏、史卷、人物关系和可选自然语言入口。
4. 新界面开发前，对照 `docs/ui-style-guide.html` 活样式指南取用现成样式；现有示例若与本规范的移动端选择交互冲突，以本规范为准。
5. 评审界面时同时检查视觉一致性与决策可读性：玩家是否看得懂自己要做什么、付出什么、冒什么风险。
